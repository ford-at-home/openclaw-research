# openclaw-research

Versioned knowledge repository for the OpenClaw AI assistant. Contains three distinct content systems in one repo.

## Content Systems

### 1. Research Library (deep async publications)

Published via `research-and-publish.sh` or the `/research` slash command. Tech-focused deep research organized by the taxonomy in `TAXONOMY.md`.

- **Directories:** `ai-ml/`, `cloud-infrastructure/`, `software-engineering/`, `web-development/`, `business-strategy/`, `community-building/`, `data-science/`, `security/`, `hardware-iot/`
- **Format:** Markdown with YAML frontmatter (`query`, `processor`, `run_id`, `date`)
- **Index:** `index.json` and `README.md` (auto-generated by `build-index.py`)
- **Template:** `_templates/research-template.md`

### 2. Personal Wiki (voice memo knowledge graph)

Built from voice memos via `ingest-voice.sh` → `wiki-source-entry.sh` → `wiki-integrate.sh`. Uses a personal domain taxonomy (family, health, work, hobbies, etc.) defined in the `wiki-integrate.sh` prompt.

- **Directories:**
  - `wiki/source/` — raw voice memo source pages
  - `wiki/entity/` — people, places, tools, orgs
  - `wiki/concept/` — recurring abstractions and patterns
  - `wiki/synthesis/` — cross-source conclusions
  - `wiki/index.md` — maintained by `wiki-integrate.sh`
  - `raw/voice/` — audio files, transcripts, metadata JSON
- **Format:** Markdown with YAML frontmatter (`page_type`, `title`, `domain`, `subdomain`, `tags`, `sources`)

### 3. Personal KB (bot-managed profiles)

Simple personal facts the bot saves directly. Not processed through either pipeline.

- **Directories:** `personal/`
- **Format:** Markdown with minimal frontmatter

### Legacy

- `voice-notes/` — older voice transcriptions from before the wiki pipeline existed. May be migrated into `wiki/source/` eventually.

## Repo Layout

- `TAXONOMY.md` — research category/subcategory definitions (9 categories, 45 subcategories)
- `index.json` / `README.md` — auto-generated (do not edit)
- `scripts/build-index.py` — research index builder
- `_templates/research-template.md` — frontmatter format reference
- `.claude/commands/research/` — slash commands for research workflows

## External Pushes

The OpenClaw bot on EC2 pushes directly to `main`. Expect the remote to move at any time. Always `git pull --rebase` before pushing to avoid conflicts.

## GitHub Actions Pipeline

`.github/workflows/index.yml` triggers on every content push to `main`:
1. Rebuilds `index.json` and `README.md` via `build-index.py`
2. Commits the updated index back (as `github-actions[bot]`)
3. Optionally notifies the EC2 agent via SSM

The workflow ignores pushes that only touch `index.json`, `README.md`, `.github/**`, or `scripts/**` to avoid loops.

## Conventions

- Research filenames: `YYYY-MM-DD-kebab-case-title.md`
- All research files must have YAML frontmatter (see `_templates/`)
- New research must land in an existing taxonomy category — if none fits, add the category to `TAXONOMY.md` first
- Never edit `index.json` or `README.md` by hand
- AWS CLI: default profile only (no `--profile` flag)
