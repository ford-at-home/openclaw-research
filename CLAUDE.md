# openclaw-research

Versioned research library for the OpenClaw AI assistant. Auto-indexed on push via GitHub Actions.

## Repo Layout

- Research files: markdown with YAML frontmatter, organized by taxonomy category or `voice-notes/YYYY/MM/DD/`
- `TAXONOMY.md` — category/subcategory definitions
- `index.json` / `README.md` — auto-generated by `.github/workflows/index.yml` (do not edit)
- `scripts/build-index.py` — index builder
- `_templates/research-template.md` — frontmatter format reference
- `.claude/commands/research/` — slash commands for research workflows

## External Pushes

The OpenClaw bot on EC2 pushes directly to `main` — voice notes, research files, and personal KB entries. Expect the remote to move at any time. Always `git pull --rebase` before pushing to avoid conflicts.

Files the bot creates:
- `voice-notes/YYYY/MM/DD/*.md` — transcribed voice memos
- `personal/*.md` — personal knowledge base entries
- Category dirs (`ai-ml/`, `software-engineering/`, etc.) — deep research

## GitHub Actions Pipeline

`.github/workflows/index.yml` triggers on every content push to `main`:
1. Extracts metadata from the pushed file (title, summary)
2. Rebuilds `index.json` and `README.md` via `build-index.py`
3. Commits the updated index back (as `github-actions[bot]`)
4. Optionally notifies the EC2 agent via SSM

The workflow ignores pushes that only touch `index.json`, `README.md`, `.github/**`, or `scripts/**` to avoid loops.

## Conventions

- Research filenames: `YYYY-MM-DD-kebab-case-title.md`
- All research files must have YAML frontmatter (see `_templates/`)
- Never edit `index.json` or `README.md` by hand — they are overwritten on every push
- AWS CLI: default profile only (no `--profile` flag)
